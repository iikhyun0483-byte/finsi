## 1. 시스템 실제 구조

### ✅ 실제로 구현된 것

#### 데이터 소스
- **Yahoo Finance API** (무료, API 키 불필요)
  - 14개 주식/ETF (SPY, QQQ, GLD, SLV, TLT 등)

  - 24시간 가격 변동률

- **Alternative.me API** (무료)
  - 암호화폐 공포탐욕지수 (0~100)

## 2. 점수 계산 방식 (실제 코드)

### Layer 1 (기술적 지표, 30%) - 최대 100점
```typescript
RSI: 0~30점 (과매도일 때 높은 점수)
MACD: 0~25점 (상승 추세일 때 높은 점수)
볼린저밴드: 0~25점 (하단 밴드 터치 시 높은 점수)
골든크로스: 0~20점 (SMA50 > SMA200일 때)
```

### Layer 2 (팩터, 40%) - 최대 100점
```typescript
12개월 모멘텀: 0~40점 (수익률 20% 이상 시 만점)
3개월 모멘텀: 0~30점 (수익률 10% 이상 시 만점)
변동성: 0~30점 (변동성 1% 미만 시 만점)
```

### Layer 3 (매크로, 30%) - 최대 100점
```typescript
공포탐욕지수: 0~30점 (25 미만일 때 만점, 역방향)
VIX: 0~25점 (15 미만일 때 만점, 역방향)
기준금리: 0~25점 (2% 미만일 때 만점, 역방향)
버핏지수: 0~20점 (100 미만일 때 만점, 역방향)
```

### 최종 점수 (가중 평균)
```typescript
finalScore = Layer1 × 0.3 + Layer2 × 0.4 + Layer3 × 0.3
```

## 4. 실제 정확도 (솔직한 평가)

### ❌ 백테스트 결과 없음
- **현재 코드에 백테스트 기능은 있지만 실행된 적 없음**
- 과거 데이터로 이 점수 시스템이 실제로 수익을 냈는지 **검증 안 됨**
- 승률, 평균 수익률, MDD 등의 통계 **없음**

### ⚠️ 신호의 실제 의미
```
"지금 사기 좋음" ≠ 수익 보장
= 단지 점수가 75점 이상이라는 뜻

점수 75점 = 기술적으로 과매도 + 모멘텀 양호 + 매크로 우호적
BUT, 이것이 미래 수익을 보장하지 않음
```

### 🔍 지표의 한계

#### 1. 후행 지표 (Lagging Indicators)
```
RSI, MACD, 이동평균 → 모두 과거 가격 기반
= 이미 오른 후에 매수 신호 나올 수 있음
= 이미 내린 후에 매도 신호 나올 수 있음
```

#### 2. 가격 데이터만 사용
```
✅ 가격, 거래량
❌ 기업 실적 (매출, 순이익)
❌ 재무제표 (PER, PBR, ROE)
❌ 뉴스, 공시
❌ 시장 심리 (개인/기관 매매 동향)
```

#### 3. 암호화폐의 특수성
```
전통적 기술적 지표가 암호화폐에 적용되면:
- 변동성이 너무 커서 신호가 불안정
- 24시간 거래로 갭(gap)이 없어 패턴이 다름
- 펀더멘털 분석 불가능 (실적이 없음)

→ 점수가 높아도 폭락 가능
→ 점수가 낮아도 폭등 가능
```

## 5. 부족한 점 (치명적 문제)

### 🚨 1. 백테스트 미실시
```
이 점수 시스템이 과거 데이터에서:
- 승률이 몇 %인지?
- 평균 수익률이 얼마인지?
- MDD(최대 낙폭)가 얼마인지?
- 샤프 비율이 얼마인지?

→ 전혀 알 수 없음
→ 신호의 신뢰도를 수치로 증명 불가
```

### 🚨 2. 팩터 점수의 불완전성
```typescript
// 현재 구현된 것
✅ 모멘텀 (12개월, 3개월)
✅ 저변동성

// 미구현 (중요한 팩터)
❌ 가치 (PER, PBR) - 저평가 종목 찾기
❌ 퀄리티 (ROE, 부채비율) - 우량 기업 찾기
❌ 사이즈 (시가총액) - 소형주 vs 대형주
❌ 배당 수익률
```

### 🚨 3. 매크로 지표의 부정확성

#### 기준금리
```typescript
// 코드: lib/macro.ts:89-94
const quote = await getYahooQuote("^IRX"); // 3개월물 국채
return quote.regularMarketPrice; // 이건 국채 수익률이지 기준금리가 아님

실제 기준금리: FRED API 필요 (환경변수 미설정 시 작동 안 함)
현재 사용 중: 3개월물 국채 수익률 (근사치)
```

#### 버핏지수
```typescript
// 코드: lib/macro.ts:116-120
const estimatedMarketCap = spy.regularMarketPrice * 120; // 추정
const gdpEstimate = 28000; // 2025년 예상 GDP (하드코딩)
const indicator = (estimatedMarketCap / gdpEstimate) * 100;

→ 실시간 GDP 데이터 없음
→ 하드코딩된 추정치 사용
→ 정확도 낮음
```

### 🚨 4. 리스크 관리 부재
```typescript
// 현재 시스템
✅ 점수 계산
✅ 매수 신호

// 미구현
❌ 손절 기준 (자동 계산 없음)
❌ 익절 기준 (자동 계산 없음)
❌ 포지션 크기 (얼마나 살지)
❌ 분할 매수/매도 전략
❌ 위험도별 포트폴리오 비중
```

### 🚨 5. 실시간성 부족
```
데이터 업데이트 주기:
- Yahoo Finance: API 호출할 때만 (수동 새로고침)
- Binance: API 호출할 때만
- 매크로 지표: API 호출할 때만

→ 실시간 모니터링 없음
→ 급등/급락 시 알림 없음 (auto-trade는 모의투자만)
→ 자동 매매 미구현 (계좌 연동 없음)
```

### 🚨 6. 종목 수 제한
```
주식/ETF: 14개만 (S&P 500 전체 아님)
암호화폐: 8개만 (수천 개 중)

→ 다양한 기회 포착 불가
→ 내가 원하는 종목 분석 불가 (분석 페이지는 검색 가능하지만)
```

## 6. 현실적인 사용법 (권장)

### ✅ 이렇게 사용하면 OK
```
1. 참고용 지표로만 활용
   - "이 종목 기술적으로 과매도네" 정도
```

### ❌ 이렇게 사용하면 위험
```

   - 현재 자동매매는 모의투자만 (실제 거래 안 됨)

3. 고위험 자산에 큰 금액
   - 암호화폐는 변동성이 매우 큼

4. 손절 없이 보유
   - 시스템이 손절 기준 제공 안 함
```

## 7. 정확도에 대한 솔직한 답변

### Q: 이 시스템 신호 따르면 수익 나나요?

**A: 모릅니다. 백테스트를 안 해봤기 때문입니다.**

```
백테스트 없음 = 과거 성과 모름
과거 성과 모름 = 미래 성과 예측 불가

단지 "기술적으로 과매도" + "모멘텀 있음" + "매크로 우호적"
이라는 사실만 알려줄 뿐
```

### Q: 점수 75점 이상이면 오를 확률이 높나요?

**A: 확률을 계산한 적이 없어서 모릅니다.**

```
백테스트로 확인 필요:
- 75점 이상 신호가 과거에 100번 나왔다면
- 그 중 몇 번이 실제로 올랐는지
- 평균 수익률이 얼마였는지

→ 이 데이터가 없음
→ 확률 제시 불가
```

### Q: 그럼 이 시스템은 쓸모없나요?

**A: 아니요. 보조 지표로는 유용합니다.**

```
✅ 유용한 점:
- 여러 지표를 한 번에 확인 가능
- 과매도/과매수 구간 파악 용이
- 매크로 환경 빠르게 체크

❌ 무용한 점:
- 신호만 믿고 투자하는 것
- 자동 매매로 방치하는 것
- 손절/익절 기준 없이 사용
```
