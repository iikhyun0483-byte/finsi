## 🔍 신호 검증이란?

### 목적
```
"점수 75점 = 실제로 X% 확률로 수익"을 검증

과거 데이터에서 점수가 높은 신호가 발생했을 때,
실제로 수익이 났는지 통계를 내서 신뢰도를 확인
```

### 작동 방식
```
1. 과거 1년 데이터 수집
2. 매일 신호 점수 계산
3. 점수 >= 기준점 이상인 날 찾기
4. 그 날 매수했다면:
   - 1주일 후 (5거래일) 수익률
   - 1개월 후 (21거래일) 수익률
   - 3개월 후 (63거래일) 수익률
5. 승률 계산 (수익 난 비율)
```

---

## 📊 표시되는 통계

### 주요 지표

#### 1. 발생 신호 수
```
점수 기준 이상인 신호가 1년간 몇 번 발생했는지
예: 75점 이상 신호가 12개 발생
```

#### 2. 승률 (%)
```
1주일 승률: 수익 난 신호 / 전체 신호 × 100
1개월 승률: 수익 난 신호 / 전체 신호 × 100
3개월 승률: 수익 난 신호 / 전체 신호 × 100

예: 12개 신호 중 8개 수익 → 승률 66.7%
```

#### 3. 평균 수익률 (%)
```
모든 신호의 평균 수익률 (손실 포함)

예: 1주일 평균 +2.5% (12개 신호의 평균)
    1개월 평균 +5.1%
    3개월 평균 +8.3%
```

## 📈 결과 해석

### 승률이 50% 이상이면?
```
✅ 좋은 신호
→ 동전 던지기(50%)보다 나음
→ 어느 정도 신뢰 가능

예: 1개월 승률 65%
→ 10번 중 6.5번은 수익
→ 참고할 가치 있음
```

### 승률이 50% 미만이면?
```
❌ 나쁜 신호
→ 동전 던지기보다 못함
→ 신호를 반대로 따라야 할 수도

예: 1개월 승률 35%
→ 10번 중 3.5번만 수익
→ 신호 신뢰도 낮음
```

### 평균 수익률 확인
```
승률이 높아도 평균 수익률이 낮으면:
→ 작은 수익 많고, 큰 손실 적음

승률이 낮아도 평균 수익률이 높으면:
→ 큰 수익 몇 번이 손실 상쇄
→ "손절은 빠르게, 수익은 오래" 전략

예:
승률 40%, 평균 +10% → 한 번 맞으면 크게 벌어서 괜찮음
승률 70%, 평균 -2% → 자주 맞지만 손실이 더 커서 안 좋음
```

// 1년치 데이터에서 각 시점마다

  }

## ⚠️ 주의사항

### 1. 백테스트 != 미래 보장
```
과거에 승률 70%였다고
미래에도 70%는 아님

시장 환경이 바뀌면 승률도 변함
```

### 2. 1년 데이터의 한계
```
1년 = 샘플 크기 작음
신호 10~20개 정도 발생

통계적으로 유의미하려면 최소 30개 필요
→ 참고만 하고, 맹신은 금물
```

### 3. 수수료/세금 미포함
```
계산된 수익률은 순수 가격 변동만
실제로는:
- 거래 수수료
- 세금
- 슬리피지 (주문 가격 차이)
등이 추가로 차감됨
```

### 4. 점수 기준에 따라 달라짐
```
75점 기준: 신호 많음, 승률 보통
90점 기준: 신호 적음, 승률 높을 수도

여러 기준으로 테스트해보세요
```

## 🎯 실전 활용법

### 1. 여러 종목 테스트
```
SPY, QQQ, GLD, BTC 등
각 종목마다 승률 확인

어떤 종목에 시스템이 잘 맞는지 파악
```

### 2. 점수 기준 최적화
```
50점, 60점, 70점, 80점, 90점
각각 승률 확인

승률이 가장 높은 기준 찾기
```

### 3. 기간 선택
```
1주일: 단기 트레이딩
1개월: 스윙 트레이딩
3개월: 중기 투자

본인 투자 스타일에 맞는 기간 선택
```

### 4. 평균 수익률 vs 승률
```
승률 높고 + 평균 수익률 높음 = 최고
승률 높고 + 평균 수익률 낮음 = 꾸준함
승률 낮고 + 평균 수익률 높음 = 한 방
승률 낮고 + 평균 수익률 낮음 = 최악
```
